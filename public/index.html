Data & methodology

The backend loads historical series and applies a monthly SIP schedule: each month, a fixed amount buys shares at a price rule you implicitly choose via calendar settings (first usable trading day on or after a target day of month where applicable). Valuation can use the latest quote date from the data source. XIRR is computed on the simulated cash flows in the currency mode you select (USD flows in US mode; INR flows in India mode, with USD shown where relevant using the same FX series).

Market data sources

  • Stock and FX history: Daily closes and metadata are requested from Yahoo Finance’s public chart API (JSON). Requests use parameters configured in the application (for example daily interval). Live screens may show more timely numbers than this tool.
  • Ticker search: Listed US names and symbols are resolved using the SEC’s published company ticker file (JSON over HTTPS). Only a subset of exchanges may be included in search results.
  • Benchmarks (e.g. broad indices or metals where implemented): Series are typically served from cached monthly files maintained with the app, with optional API fill when a month is missing—see operator documentation for sync scripts.
  • USD/INR for India mode before Yahoo’s INR=X history (~Dec 2003): The server stitches a monthly USD/INR series (EXINUS-style CSV: observation date + rate) with Yahoo’s daily INR/USD history where they meet. File location is configurable (data/exinus-monthly.csv or EXINUS_CSV_PATH).

Limitations

  • Data providers can change, delay, or correct data; outages and rate limits can cause errors or partial results.
  • The model is simplified: it does not replicate every real-world portfolio rule, tax treatment, or corporate-action nuance unless reflected in the underlying series.

Privacy

This page is designed to minimize personal data: there is no login, no profile, and no portfolio database stored by the application for you.

What stays on your device

  • Investor mode (USD vs INR): Your toggle choice may be stored in the browser’s localStorage under a small app-specific key so the page can reopen in the same mode. You can clear site data in your browser to remove it.
  • Cookies: This tool does not rely on advertising or analytics cookies for core functionality; behavior depends on how the public site is hosted (third-party scripts, if any, are outside this repository unless you add them).

What the server may see

  • API requests: When you run estimates, the browser calls this app’s own API. The server sees normal HTTP data (for example path, method, rough timing). To limit abuse, requests may be rate-limited per IP address (using forwarded headers such as X-Forwarded-For when present).
  • Logging: Operators can configure stdout logging and optional file logging via LOG_LEVEL and LOG_FILE. Logs may include technical details useful for debugging (for example errors and request metadata). They are not intended to build marketing profiles.
  • Retention & sharing: Log retention and hosting subprocessors depend on where you deploy (e.g. Railway or another provider). This open-source app does not sell your data; any broader policy is for the site operator to publish if needed.
